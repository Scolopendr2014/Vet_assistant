# Конвенция доступа к зависимостям (DI) в presentation

Документировано по результатам спринта 9 (VET-185, VET-188).

## Правило

- **В слое presentation (страницы, виджеты, провайдеры UI):** доступ к репозиториям, сервисам и use case'ам — **только через провайдеры Riverpod** (например, `ref.read(patientRepositoryProvider)`), а не через прямой вызов `getIt<...>()`.
- **Регистрация** зависимостей выполняется в **GetIt** в `lib/core/di/di_container.dart`.
- **Провайдеры** в фичах (например, `patient_providers.dart`, `stt_providers.dart`) оборачивают `getIt<...>()` и экспортируют зависимость в UI. Так обеспечиваются единый способ доступа и возможность подмены в тестах через `ProviderScope`/override.

## Исключения

- **Точка входа приложения** (`main.dart`) может получать из GetIt только минимально необходимое (например, `AppRouter`).
- **Навигация** (AppRouter) получает зависимости через конструктор (инжекция из GetIt при регистрации), без прямого вызова `getIt` в коде роутера.

## Итог

GetIt — контейнер регистрации; в коде страниц и виджетов используется только Riverpod (провайдеры). Новые зависимости для UI регистрируются в GetIt и при необходимости экспортируются провайдером в соответствующей фиче.
