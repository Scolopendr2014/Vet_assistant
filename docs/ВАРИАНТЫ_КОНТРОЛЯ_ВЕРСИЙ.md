# Варианты контроля версий при ограниченной памяти (8 ГБ ОЗУ)

На ноутбуке с 8 ГБ ОЗУ полный **GitLab в Docker** (требует 6–8 ГБ только на контейнер) обычно не подходит. Ниже — рабочие варианты.

---

## 1. Git локально + облачный хостинг (рекомендуется)

**Суть:** на ноутбуке только **Git** (без своего сервера). Репозитории хранятся и в папке проекта, и в облаке (GitHub, GitLab.com и т.п.).

**Плюсы:** почти не тратит память, бесплатные тарифы, бэкапы в облаке, удобно для команд и портфолио.  
**Минусы:** нужен интернет для push/pull (локально коммиты работают без сети).

**Что сделать:**
1. Установить [Git для Windows](https://git-scm.com/download/win).
2. Зарегистрироваться на [GitHub](https://github.com) или [GitLab.com](https://gitlab.com) (бесплатно).
3. **Для GitLab.com:** пошаговая настройка проекта — [docs/НАСТРОЙКА_GITLAB_COM.md](НАСТРОЙКА_GITLAB_COM.md).
4. В папке проекта (если репозиторий ещё не подключён):
   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   git remote add origin https://github.com/ВАШ_ЛОГИН/ИМЯ_РЕПОЗИТОРИЯ.git
   git push -u origin main
   ```
5. Дальше работаете как обычно: `git add`, `git commit`, `git push`, `git pull`. Всё описание в [docs/НАСТРОЙКА_GIT.md](НАСТРОЙКА_GIT.md).

**Память:** только работа Git (десятки МБ).

---

## 2. Лёгкий свой сервер: Gitea в Docker

**Суть:** свой веб-интерфейс для Git (репозитории, веб-UI, базовый CI), но в разы легче GitLab.

**Плюсы:** всё на своём компьютере, открытый исходный код, ~200–500 МБ ОЗУ.  
**Минусы:** меньше возможностей, чем у GitLab (нет встроенного полноценного CI как в GitLab, но есть базовые вещи).

**Пример запуска (один контейнер):**
```bash
docker run -d --name gitea -p 3000:3000 -v gitea_data:/data gitea/gitea:latest
```
Открыть в браузере: http://localhost:3000, при первом заходе — быстрая настройка (создать админа, имя сервера и т.д.).

**С Docker Compose** (удобнее для остановки/запуска) можно добавить в проект отдельный `docker-compose.gitea.yml` или заменить текущий `docker-compose.yml` на сервис Gitea — при необходимости могу подготовить пример файла.

**Память:** ориентировочно 200–500 МБ.

---

## 3. Только Git, без веб-интерфейса

**Суть:** на ноутбуке только Git. Никакого своего сервера и веб-UI.

**Плюсы:** минимум ресурсов, простота, полный контроль версий в терминале и в IDE.  
**Минусы:** нет своего веб-интерфейса (можно смотреть историю в IDE или на GitHub/GitLab.com после push).

**Что сделать:** установить Git, в папке проекта выполнить `git init`, делать коммиты. При желании один раз настроить remote на GitHub/GitLab.com и делать `git push` для бэкапа и совместной работы. Подробности — в [docs/НАСТРОЙКА_GIT.md](НАСТРОЙКА_GIT.md).

**Память:** практически не влияет на ОЗУ.

---

## 4. Облачные IDE с встроенным Git

**Суть:** разработка в браузере (GitHub Codespaces, GitLab Web IDE, Replit и т.п.) — Git и окружение уже в облаке.

**Плюсы:** не нагружают ваш ноутбук, Git и сборки — на стороне сервиса.  
**Минусы:** зависимость от интернета и правил сервиса, бесплатные лимиты.

Подходит как дополнение, а не замена локального Git: основной код можно вести локально (вариант 1) и при необходимости открывать в облачной IDE.

---

## Сравнение

| Вариант                    | ОЗУ     | Свой сервер | Веб-интерфейс | Сложность |
|---------------------------|---------|-------------|----------------|-----------|
| Git + GitHub/GitLab.com   | Минимум | Нет         | На сайте       | Низкая    |
| Gitea в Docker            | ~300 МБ | Да          | Да             | Средняя   |
| Только Git локально       | Минимум | Нет         | Нет            | Низкая    |
| GitLab в Docker            | 6–8 ГБ  | Да          | Да             | Высокая   |

---

## Рекомендация для 8 ГБ ОЗУ

1. **Основной вариант:** **Git + GitHub или GitLab.com** (вариант 1). Контроль версий полный, память почти не тратится, удобно и для себя, и для команды.
2. **Если нужен именно свой сервер с веб-интерфейсом на ноутбуке:** **Gitea в Docker** (вариант 2) — заменит GitLab по базовым сценариям без перегрузки памяти.

Если нужно, могу подготовить в проекте пример `docker-compose.gitea.yml` и короткую инструкцию по первому запуску Gitea в вашей папке.
