# Запуск сборки: путь без кириллицы

На Windows сборка Android (Gradle, компилятор шейдеров Impeller) и запись файлов в `build/` часто ломаются, если путь к проекту содержит **кириллицу** (например «Новая папка», «Асистент ветеринара»).

Ошибки при этом могут быть такие:
- `Could not write file to ...\build\...\shaders\stretch_effect.frag`
- `ShaderCompilerException` при сборке
- Раньше: `Your project path contains non-ASCII characters`

## Решение: перенести проект в папку только с латиницей

1. **Закройте проект в Cursor/IDE** (чтобы не было блокировки файлов).

2. **Скопируйте всю папку проекта** в каталог с путём только из латинских букв, например:
   - `C:\Users\User\Desktop\vet_assistant`
   - или `C:\dev\vet_assistant`
   - или `C:\projects\vet_assistant`

   Не переименовывайте вложенные папки вручную — копируйте **целиком** папку «Асистент ветеринара» в новое место и при желании переименуйте уже **эту** целевую папку (например в `vet_assistant`).

3. **Откройте проект из нового пути** в Cursor (File → Open Folder → выберите `C:\...\vet_assistant`).

4. В терминале в корне проекта выполните:
   ```bash
   flutter clean
   flutter pub get
   flutter run
   ```
   (или `flutter run -d RZCX80KSX1D` для вашего устройства.)

После этого сборка и запись шейдеров в `build/` должны проходить без ошибок.

## Если хочется оставить исходную папку

Можно оставить копию в «Новая папка» для бэкапа, но **собирать и запускать** приложение нужно из папки с путём без кириллицы. При необходимости настройте Git так, чтобы основная работа велась из новой папки (один и тот же репозиторий можно клонировать в `C:\dev\vet_assistant`).
