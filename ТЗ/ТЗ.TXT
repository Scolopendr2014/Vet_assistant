Техническое задание: приложение «Протокол ветеринарного осмотра»
1. Общее описание
Мобильное приложение для ветеринарных врачей, позволяющее формировать протоколы осмотра по специальностям: стоматология, кардиология, УЗИ. Работает автономно на смартфоне, не требует постоянного подключения к сети.
2. Целевая аудитория
Ветеринарные врачи всех специализаций, работающие в клиниках или на выезде.
3. Основные функции
3.1. Блок регистрации пациентов
Поля для заполнения:
вид животного (вид);
пол (пол);
кличка (кличка);
порода (порода);
окрас (окрас);
чип (номер чипа);
тату (описание тату);
ФИО владельца (ФИО владельца);
телефон владельца (телефон);
email владельца (email).
Ограничения:
бесплатная версия — до 10 пациентов;
платная версия — неограниченное количество.
3.2. Блок «Протокол осмотра»
Кардиология:
список параметров в виде скролла (выпадающих списков, чекбоксов, полей ввода);
обязательные поля: ЧСС, ритм, тоны, шумы, АД, данные ЭКГ;
кнопка «Сформировать протокол» → генерация PDF.
УЗИ:
аналогичная структура с кардиологией;
параметры по системам органов (сердце, печень, почки и т. д.);
поля для описания эхогенности, размеров, структур;
кнопка «Сформировать протокол» → генерация PDF.
Стоматология:
графическая стоматологическая карта (схема зубов с возможностью отметки состояний);
легенда состояний (кариес, перелом, отсутствие и т. п.);
текстовые поля для описания прикуса, состояния дёсен;
кнопка «Сформировать протокол» → генерация PDF.
3.3. Хранение данных
локальное хранение протоколов в формате PDF на устройстве;
поиск по пациентам (по кличке, виду, дате осмотра);
сортировка протоколов по дате, виду животного, специальности.
3.4. Экспорт и отправка
сохранение протокола в PDF на устройство;
отправка PDF на email владельца (из данных пациента);
возможность поделиться файлом через мессенджеры/облако.
4. Требования к интерфейсу
интуитивно понятный дизайн;
адаптивность под экраны смартфонов (портретная ориентация);
цветовая схема: нейтральная (белый/серый фон, контрастный текст);
иконки для быстрого доступа к блокам («Пациенты», «Протоколы», «Настройки»).
5. Технические требования
платформы: Android, iOS;
язык разработки: Kotlin (Android), Swift (iOS) или кросс‑платформенный фреймворк (Flutter, React Native);
локальная база данных: SQLite или аналоги;
экспорт в PDF: встроенная библиотека генерации PDF;
безопасность: шифрование локальных данных (опционально).
6. Платные функции (премиум‑версия)
неограниченное количество пациентов;
облачное хранилище протоколов;
шаблоны протоколов для дополнительных специальностей (неврология, офтальмология и т. д.);
мультипользовательский доступ (для клиник).
7. Этапы разработки
Прототипирование интерфейса (1–2 недели).
Разработка ядра приложения (регистрация, база данных) (3–4 недели).
Реализация блоков протоколов (кардиология, УЗИ, стоматология) (4–6 недель).
Тестирование и отладка (2–3 недели).
Публикация в App Store и Google Play (1 неделя).
8. Критерии приёмки
корректная работа на устройствах с ОС Android 8.0+ и iOS 13+;
отсутствие критических ошибок при формировании PDF;
время отклика интерфейса < 0,5 сек;
объём занимаемой памяти < 100 МБ (без учёта сохранённых PDF).
